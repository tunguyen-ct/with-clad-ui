## Scale Naming

When it comes to name the different sizes and weights in scales, we'll use following naming convention:

- ...
- **xxs**
- **xs** (extra small)
- **sm** (small)
- **md** (medium)
- **lg** (large)
- **xl** (extra large)
- **xxl**
- **xxxl**
- ...

## Spacing

```
theme.space.[ none | xxxs | xxs | xs | sm | md | lg | xl | xxl | xxxl | xxxxl | xxxxxl ];
```

Use spacings for gutters, margins, and paddings. Don't use it for border width, border radius, icon size, font size, or line height. Use the dedicated theme properties instead.

## Sizes

```
// absolute scale steps
theme.sizes.[ xxxs | xxs | xs | sm | md | lg | xl | xxl | xxxl | xxxxl | xxxxxl | buttonMaxWidth | $sm | $md | $lg | $xl ]

// percentage and relative to screen size
theme.sizes.[ quarter | third | half | twoThird | threeQuarter | full | vW | vH ];
```

Use sizes for width, height, min-width, max-width, min-height and max-height.

## Durations

```js
theme.durations.[ xxxs | xxs | xs | sm | md | lg | xl | xxl | xxxl | xxxxl ];
```
**Note**: `duration` is not (yet) a standard System UI Theme Specification, therefore, it will not work with `sx()` helper. (Access the tokens via `theme` object instead).
